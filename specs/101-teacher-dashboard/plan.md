## Summary

The Teacher Dashboard will be a comprehensive command center for educators to manage their classes, students, and academic workflows. It will integrate existing backend functionalities for attendance, assignments, and results while introducing a new "Student Monitoring" view that allows teachers to see a student's performance summary (similar to the student dashboard). The primary technical approach involves building a highly interactive, responsive dashboard using Tailwind CSS and custom web components, backed by a robust Express/Mongoose API.

## Technical Context

**Language/Version**: TypeScript, Node.js (ESM)
**Primary Dependencies**: Express, Mongoose, Socket.io, Chalk
**Storage**: MongoDB (Mongoose)
**Testing**: Jest, Supertest
**Target Platform**: Web (Vercel compatible)
**Project Type**: Web application (Fullstack)
**Performance Goals**: Dashboard load < 1.5s; Grading updates < 500ms.
**Constraints**: Must maintain role-based access control (RBAC); reuse existing project components.
**Scale/Scope**: ~10 teacher-specific views; handling thousands of student records.
