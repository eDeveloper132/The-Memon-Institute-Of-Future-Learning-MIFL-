## Summary
Implement a Parent Management system for Admins. This includes a new backend endpoint for linking parents to multiple students and a new `parents.html` UI in the admin dashboard. We will leverage the existing `User` model's `role: 'parent'` and the student's `parent` field.

## Technical Context

**Language/Version**: TypeScript / Node.js 20+
**Primary Dependencies**: Express, Mongoose, Socket.IO
**Storage**: MongoDB
**Testing**: Manual verification + unit tests (existing pattern)
**Target Platform**: Vercel (Production) / Local Server
**Project Type**: Web application (Express backend + Vanilla JS frontend)
**Performance Goals**: <200ms for linking operations
**Constraints**: Must maintain consistency between `student.parent` (ID) and `student.parentName`/`parentContact` (redundant strings).
**Scale/Scope**: ~1k parents, ~5k students.
